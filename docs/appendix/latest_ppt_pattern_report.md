## 4. 관찰된 슈카월드식 PPT 문법

### 4.1 “한 장에 한 메시지”가 강하다

대형 덱도 한 슬라이드에 여러 논점을 몰아넣지 않는다. `국민도 주주가 되는가`는 114장짜리지만, 실제 문법은 원문 한 문장 또는 질문 하나를 여러 장에 나눠서 쌓는 방식이다. `전당포 주식회사`도 55장 중형 덱이지만 F88 성장 숫자, 한국 전당포 기억, 베트남 금융구조, 오토바이 담보대출, 창업자 서사를 각각 섹션으로 나눠 처리한다.

### 4.2 섹션 전환은 “내용 구분”보다 “방송 호흡”에 가깝다

섹션 후보 슬라이드는 대체로 본론 전환점이다.

- `전당포 주식회사`: 2, 11, 18, 43
  - F88 소개
  - 한국의 전당포 이미지
  - 서민들의 자금줄 “오담대”
  - F88 성장의 그림자
- `코카콜라를 이기는 방법`: 2, 39, 55
  - 코카인 하마
  - 암바니/콜라 시장 전환
  - 회수/마무리
- `요즘 뜨는 레이저 치료`: 3, 9, 24, 52, 59, 77, 85
  - 드론 비용 문제를 여러 단계로 쪼갠 후 한국/피부과 punchline으로 회수

### 4.3 notes는 `piti`의 핵심 계약이다

PPT 본문만 읽으면 출처 관리가 누락된다. `piti`는 슬라이드 본문과 speaker notes를 분리해서 생성해야 한다.

권장 slide object:

```json
{
  "headline": "...",
  "body": ["..."],
  "source_urls": ["..."],
  "image_urls": ["..."],
  "notes": "[내용] ...\n[이미지] ..."
}
```

### 4.4 이미지 출처 형식은 통일되어 있지 않다

확인된 형식:

```text
[이미지] URL
이미지 - URL
사진: URL
GPT 생성
```

따라서 parser와 `piti`는 bracket label만 기대하면 안 된다. 지금 parser의 loose label 보정은 유지해야 한다.

## 5. piti 구현 시사점

1. 첫 버전은 “예쁜 완성본”보다 notes 포함 draft deck이어야 한다.
2. slide type은 최소한 `title`, `section_title`, `text`, `quote`, `data`, `image_placeholder`, `closing_question`으로 나누면 된다.
3. 이미지 자동 삽입보다 image placeholder + notes 출처 삽입이 우선이다.
4. 김동찬/김성원 덱은 이미지 중심/짧은 문장 중심이 많아, visible text만으로 slide_type을 판정하면 `image_centered`가 과다 집계된다. 추후 layout 정보와 media count를 같이 써야 한다.
5. parsed PPT를 그대로 prompt에 모두 넣기보다, 섹션별 5~10장 샘플을 retrieval해서 쓰는 방식이 필요하다.

## 6. 다음 보정 과제

- section_title heuristic을 텍스트 길이뿐 아니라 slide layout, shape count, media count로 보정
- notes source label을 표준화해 `source_urls`와 `image_urls`가 항상 안정적으로 분리되게 하기
- `전당포 주식회사`, `코카콜라를 이기는 방법`을 `anny_storyline` fixture로 재구성
