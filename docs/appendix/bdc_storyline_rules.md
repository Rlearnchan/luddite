# Appendix. BDC Storyline Rules

작성일: 2026-05-16
상태: v0.1 draft

## 1. 목적

BDC(브랜디드 콘텐츠)는 일반 방송과 비슷한 포맷을 쓰지만 목적이 다르다. Luddite MVP에서는 BDC를 일반 방송과 분리하고, 추후 별도 모드로 확장한다.

## 2. 일반 방송과 다른 점

| 항목 | 일반 방송 | BDC |
|---|---|---|
| 목표 | 흥미로운 사회/경제/시장 이야기 | 광고주 서비스/산업 이슈 자연 연결 |
| 결말 | 질문, punchline, 리스크 | 광고주 소구 포인트 설명 |
| 리스크 | 사실/저작권/민감성 | 사실/저작권/민감성 + 광고주 이해관계 |
| 수정 포인트 | 대표/리서처 말맛 | 광고주 요구사항, 서비스 설명 정확성 |

## 3. BDC 기본 구조

```text
1. 광고 서비스와 무관해 보이는 최근 산업/사회 이슈로 시작
2. 해당 산업의 구조적 문제를 설명
3. “그래서 이런 서비스/해결책이 필요하다”로 연결
4. 광고주 서비스의 기능/차별점/사용법을 자세히 설명
5. 과장 없이 한계/주의사항도 정리
```

## 4. 좋은 BDC 연결

좋은 연결:

```text
최근 이슈 → 시장 구조 → 소비자/기업의 불편 → 서비스 필요성
```

나쁜 연결:

```text
기사 3장 설명 → 갑자기 광고주 상품 소개
```

## 5. BDC에서 anny가 주의할 점

- 광고주 서비스 설명은 일반 방송보다 더 자세해야 한다.
- 대표가 새로 손보지 않도록 기능/용어/절차를 명확히 쓴다.
- 광고처럼 보이지 않는 도입부가 중요하다.
- 그러나 결말에서는 광고주 소구 포인트를 흐리지 않는다.
- 법적/의학적/투자성 표현은 더 보수적으로 쓴다.

## 6. BDC에서 piti가 주의할 점

- 일반 PPT 포맷은 유지하되 후반부 서비스 설명은 밀도가 높아질 수 있다.
- 캡처/서비스 화면/로고 사용 권한을 별도 확인한다.
- speaker notes에는 광고주 제공 자료와 외부 자료를 구분한다.

## 7. MVP 처리

초기 Luddite에서는 BDC를 다음과 같이 처리한다.

```text
- jibi: bdc_candidate flag만 붙임
- anny: 일반 storyline과 분리된 BDC outline 생성은 후순위
- piti: 일반 deck 생성 가능하되 BDC 품질 보장은 하지 않음
```

## 8. 추가로 읽을 자료

```text
- 젠스파크(BDC)
- 타입캐스트 BDC
- 결혼해듀오 BDC
- 광고주 미팅 회의록 중 리서치 관련 부분
- BDC PPT 샘플 3~5개
```
