# Appendix. Google Sheet Insights

작성일: 2026-05-16
상태: v0.1 draft

## 1. 목적

Google Sheet `슈카월드_리서치의 사본`은 Luddite에서 가장 직접적인 운영 데이터다. 특히 `주제 찾기`는 jibi의 positive/negative example, `리서치 인수인계`는 리서치팀의 판단 기준, 금칙, 선호 소재를 담고 있다.

## 2. 확인된 시트

| 시트 | 활용 우선순위 | 용도 |
|---|---:|---|
| `주제 찾기` | 1 | 후보 seed, 제작 여부, 방송 활용 여부, 채택/미사용 이유 |
| `리서치 인수인계` | 1 | 리서치 기준, 기피/선호 주제, 참고 사이트, 이미지 주의사항 |
| `방송 주제` | 2 | 최종 영상 제목/주제 변환 학습 |
| `작업 현황` | 2 | 실제 업무 프로세스와 병목 파악 |
| `슈카월드 채널` | 2 | 과거 영상 메타데이터/성과 분석 |
| `쇼츠 제작` | 3 | 본편에서 쇼츠로 넘어가는 기준. MVP에서는 후순위 |

## 3. `주제 찾기`에서 보이는 구조

초기 행 기준으로 컬럼은 다음 성격을 가진다.

```text
주제명 / 링크 / 제작 여부 / 방송 활용 여부 / 이유(채택/사용/미사용)
```

이 시트는 jibi의 학습 데이터로 특히 유용하다.

- `제작 여부 = O`, `방송 활용 여부 = O`: strong positive
- `제작 여부 = O`, `방송 활용 여부 = X`: 제작은 했으나 방송에서 죽은 예시
- `제작 여부 = X`: seed 단계에서 약했거나 우선순위가 밀린 예시
- `이유`: 어떤 판단으로 채택/미사용됐는지 직접 설명하는 귀한 annotation

## 4. jibi에 줄 수 있는 인사이트

### 4.1 방송에 살아나는 seed

`주제 찾기`의 채택 사례를 보면 단순 화제성보다 다음 조건이 중요하다.

```text
- 슈카의 기존 래퍼토리와 연결된다.
- 농업의 기업화, 효율화, 시장 구조, 국가 비교처럼 익숙한 큰 틀이 있다.
- 단발성 기사라도 뒤에 산업/제도/사람 이야기가 붙는다.
```

예: “중국 돼지 아파트”는 단순 이색 뉴스가 아니라 농업의 기업화/효율화라는 기존 래퍼토리와 연결되어 활용됐다.

### 4.2 방송에서 죽는 seed

미사용 이유에서 반복될 가능성이 있는 패턴:

```text
- 서브급이라 메인 분량으로 확장하기 어렵다.
- 실제 사례 하나가 너무 강해 특정 기업 홍보/비판처럼 보인다.
- 패턴이 너무 뻔하다.
- 소재는 재미있지만 storyline이 쉽게 떠오르지 않는다.
- 민감 소재라 활용 가능성이 낮다.
```

jibi는 이 이유들을 negative label로 사용해야 한다.

## 5. `리서치 인수인계`에서 보이는 판단 기준

이 시트에는 다음 범주가 있다.

```text
- 리서치 자료 기본 양식
- 리서치팀 공유 계정/구독 현황
- 대표 보고 관련
- 주제 선정 시 참고 내용
- 활용 사이트
- 사진 및 이미지 사용 시 주의사항
- BDC 구성
- 참고 영상 목록
```

중요: 이 시트에는 민감한 계정/비밀번호/인증 정보가 포함될 수 있으므로 LLM에 원문을 넣지 않는다.

## 6. 주제 기피/선호 기준

문서화가 필요한 범주:

### 기피 주제

```text
- 특정 인종/민족/국가/종교를 직접 비판하는 소재
- 특정 기업을 띄워주거나 공격하는 소재
- 자격증/라이선스 직군 비판
- 역사 소재 중 지적 가능성이 높은 것
- 스포츠 중심 소재
- 너무 따끈따끈한 신제품/예상성 소재
```

### 선호 주제

```text
- 공룡
- 인도
- 농구/야구는 선호 소재로 언급되나, 스포츠 자체는 조회수 리스크가 있으므로 사용 방식 주의
- 영국 왕실
- 뜬금없이 올라가는 K-국장 조롱
```
